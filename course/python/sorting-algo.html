Lesson 8.1

Linear Searching

Concept & Approach

Linear search is the most basic searching algorithm. It sequentially checks each element of the list until a match is found or the whole list has been searched. Approach: Start at the first index (0). Compare the current item with the target. If they match, return the index. If not, move to the next item. If you reach the end of the list without finding the target, return a signal value (like `-1` or `None`).

Python Implementation

def linear_search(arr, target):
    """
    Searches for a target value within a list sequentially.
    Returns the index if found, otherwise returns -1.
    """
    for i in range(len(arr)):
        if arr[i] == target:
            return i  # Target found, return index
            
    return -1  # Target not in list

Analysis

TIME COMPLEXITY O(n)

In the worst-case scenario, the target is at the very end of the list or not present at all, requiring checking all n elements.

SPACE COMPLEXITY O(1)

Uses a constant amount of extra memory regardless of the input size.

Pros

  • Extremely simple to implement.
  • Works on unsorted data.
  • Efficient for very small datasets.

Cons

  • Very slow for large datasets.
  • Inefficient if multiple searches are needed on the same data.
Lesson 8.2

Binary Searching

Concept & Approach

Binary search is a highly efficient "divide and conquer" algorithm that finds the position of a target value within a sorted array. Approach: Compare the target value to the middle element of the array. If they match, you're done. If the target is smaller, repeat the search on the left half. If the target is larger, repeat on the right half. The search space is halved in every step.

Python Implementation

def binary_search(arr, target):
    """
    Searches a sorted list using divide and conquer.
    """
    low = 0
    high = len(arr) - 1

    while low <= high:
        mid = (low + high) // 2  # Find the middle index
        
        if arr[mid] == target:
            return mid           # Target found
        elif arr[mid] < target:
            low = mid + 1        # Discard left half
        else:
            high = mid - 1       # Discard right half
            
    return -1                    # Target not found

Analysis

TIME COMPLEXITY O(log n)

Because the search area is halved with every iteration, it takes at most log₂(n) steps to find the element.

SPACE COMPLEXITY O(1)

Iterative implementation requires only a few pointer variables (`low`, `high`, `mid`).

Pros

  • Incredibly fast for large datasets.
  • Scales beautifully (searching 1 million items takes max ~20 steps).

Cons

  • Requires the data to be sorted first.
  • Requires fast random access (arrays, not linked lists).

Selection Sort

Concept & Approach

Selection sort divides the list into two parts: a sorted front portion and an unsorted back portion. Approach: Initially, the sorted part is empty. Iterate through the unsorted part to find the absolute minimum value. Swap this minimum value with the first element of the unsorted part. This element is now considered part of the sorted portion. Repeat until fully sorted.

Python Implementation

def selection_sort(arr):
    """
    Sorts an array by repeatedly finding the minimum element 
    from the unsorted part and putting it at the beginning.
    """
    n = len(arr)
    
    for i in range(n):
        # Assume the current element is the minimum
        min_idx = i
        
        # Search the rest of the array for a smaller element
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
                
        # Swap the found minimum element with the first element
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
        
    return arr

Analysis

TIME COMPLEXITY O(n²)

Contains two nested loops. Even if the array is already sorted, it still scans the remaining elements to verify the minimum.

SPACE COMPLEXITY O(1)

It is an "in-place" sorting algorithm, requiring no extra memory proportional to input size.

Pros

  • Minimizes the number of swaps (max O(n) swaps), good when memory writing is costly.
  • Simple conceptual model.

Cons

  • Very inefficient on large arrays.
  • Not an "adaptive" sort (it doesn't exit early if sorted).

Insertion Sort

Concept & Approach

Insertion sort builds the final sorted array one item at a time. It behaves much like how you might sort playing cards in your hands. Approach: Assume the first element is already sorted. Pick the next element (the "key"). Compare the key backwards against the sorted elements. Shift the sorted elements to the right until you find the correct position for the key, then insert it.

Python Implementation

def insertion_sort(arr):
    """
    Sorts an array by repeatedly picking an element and 
    inserting it into its correct position in the sorted part.
    """
    # Traverse from 1 to len(arr)
    for i in range(1, len(arr)):
        key = arr[i]
        
        # Move elements of arr[0..i-1], that are greater than key,
        # to one position ahead of their current position
        j = i - 1
        while j >= 0 and key < arr[j]:
            arr[j + 1] = arr[j]
            j -= 1
            
        arr[j + 1] = key
        
    return arr

Analysis

TIME COMPLEXITY O(n²)

Worst/Average case is O(n²). However, Best case is O(n) if the array is already mostly sorted.

SPACE COMPLEXITY O(1)

Like selection sort, it operates completely in-place.

Pros

  • Extremely efficient for small arrays or "nearly sorted" data.
  • Stable sort (maintains relative order of equal items).

Cons

  • Poor performance on large, randomly distributed datasets.
  • Terrible performance on reverse-sorted data.
Lesson 8.5

Merge Sort

Concept & Approach

Merge Sort is an advanced, recursive "divide and conquer" algorithm. Approach: First, recursively divide the list into equal halves until you have sublists containing 1 item (a single item is technically sorted). Then, repeatedly merge these sorted sublists back together to produce newly sorted sublists until only one sorted list remains.

Python Implementation

def merge_sort(arr):
    if len(arr) > 1:
        mid = len(arr) // 2
        left_half = arr[:mid]
        right_half = arr[mid:]

        # Recursive call on each half
        merge_sort(left_half)
        merge_sort(right_half)

        # Merge the sorted halves
        i = j = k = 0
        while i < len(left_half) and j < len(right_half):
            if left_half[i] < right_half[j]:
                arr[k] = left_half[i]
                i += 1
            else:
                arr[k] = right_half[j]
                j += 1
            k += 1

        # Check for any remaining elements
        while i < len(left_half):
            arr[k] = left_half[i]
            i += 1
            k += 1

        while j < len(right_half):
            arr[k] = right_half[j]
            j += 1
            k += 1
            
    return arr

Analysis

TIME COMPLEXITY O(n log n)

Guaranteed O(n log n) in best, worst, and average cases. Halving takes log(n) levels, and merging takes O(n) work per level.

SPACE COMPLEXITY O(n)

Requires additional memory to hold the temporary split sublists during the merging process.

Pros

  • Excellent and consistent performance on large datasets.
  • Stable sort.

Cons

  • Consumes O(n) extra space, which can be problematic for massive arrays in constrained environments.
  • Slight overhead for recursive function calls.
Lesson 8.6

Built-in Sorting

Concept & Approach

In the real world, you rarely write your own sorting algorithms. Python provides highly optimized, built-in sorting capabilities based on an algorithm called Timsort (a brilliant hybrid of Merge Sort and Insertion Sort designed to perform exceptionally well on real-world data).

Python Implementation

# Method 1: list.sort()
# Modifies the original list directly (in-place)
my_list = [5, 2, 9, 1, 5, 6]
my_list.sort() 
# my_list is now [1, 2, 5, 5, 6, 9]

# Method 2: sorted(iterable)
# Returns a NEW sorted list, leaves original untouched
data = (3, 1, 4, 1, 5) # Works on tuples/iterables too
new_sorted_list = sorted(data)
# new_sorted_list is [1, 1, 3, 4, 5]

# Optional arguments (Reverse & Custom Key)
words = ["apple", "banana", "kiwi", "cherry"]
words.sort(reverse=True)            # Sort descending
sorted_by_len = sorted(words, key=len) # Sort by string length

Analysis

TIME COMPLEXITY O(n log n)

Worst/Average case is O(n log n). The Best case (if data is already mostly sorted) is an incredibly fast O(n).

SPACE COMPLEXITY O(n)

Timsort uses auxiliary memory (up to n/2) to achieve its speed.

Pros

  • Professionally implemented in C (extremely fast).
  • Stable sort and very adaptive.
  • Zero risk of implementation bugs on your part.

Cons

  • Hides algorithmic mechanics (bad for learning, great for production).
